# 名片工坊-图片渲染版

这是一套真正朝“图片名片”方向走的工作流版本。

## 这版和之前的区别

之前那版：

1. 只能输出文字预览
2. 最后给用户的是文本或 HTML
3. 不会真正生成 PNG / JPG

这版：

1. 先对话采集资料
2. 生成名片预览
3. 用户确认
4. 生成 HTML 模板
5. 调用图片渲染接口
6. 返回图片链接
7. 在 Dify 中展示图片并提供下载链接

## 你必须修改的地方

导入后，请打开节点：

`图片渲染接口`

把里面的 URL：

`https://your-public-render-api.example.com/render-card`

替换成你自己的公网渲染服务地址。

## 渲染接口的要求

建议接口接收：

```json
{
  "html_content": "<html>...</html>",
  "output_filename": "business_card.png"
}
```

建议接口返回：

```json
{
  "image_url": "https://你的域名/cards/business_card.png"
}
```

## 如果你不替换接口地址会怎样

如果不替换，这个工作流虽然能导入，但到最后出图时一定失败，因为里面现在只是占位地址。

## 当前最适合的用途

1. 你已经有自己的 HTML 转图片服务
2. 你准备让开发同事补一个渲染接口
3. 你要先把 Dify 主流程搭起来，再接图片下载
